Staging API Version 1.0.0
December 07 2023

Our first timeline release! ðŸŽ‰

Important Update: Effective December 7, 2023, several significant changes have been introduced across all API endpoints. A notable modification requires that requests now include Authentication Headers with a valid JWT token, along with the introduction of a new JWT token login endpoint. Before utilizing API version v1.0.0, we strongly recommend reviewing the comprehensive API documentation available here and thoroughly examining the corresponding release notes sections within this document.

Security Fixes

Fixes

Starting December 7, 2023, a JSON Web Token (JWT) has been integrated into all APIs. A valid JWT token is now mandatory for successful API consumption, addressing a security concern associated with an insecure API endpoint.

New

In light of this update, a new login endpoint has been introduced. Users can utilize this endpoint to obtain a JWT token, which serves as the required Authentication Header for every subsequent API endpoint interaction. For detailed instructions on the proper usage of the login endpoint, please refer to the API documentation.

Enhancement

Additionally, a Request Rate Limiter has been implemented across all API endpoints. This feature allows for a maximum of 100 requests per minute per endpoint, enhancing overall system stability and performance.

Bug Fixes

Fixes

Enhanced the flexibility of the event type property in the `add guest to event` endpoint, allowing it to accept a broader range of values beyond just GUEST and VIP.

Fixes

Standardized success and error responses across all endpoints, ensuring the use of universally recognized codes for success and error messages. Additionally, implemented clear and definitive error messages for improved understanding and troubleshooting.

Changes

Fixes

Changes to add event Endpoint: The event_entrance_gate is no longer required in the request body for the add event endpoint. Please ensure that it is not included in your request.

New

Enhancement in add guest to event Endpoint: The attendee_id field from the add guest to event endpoint now returns a secure hash string instead of a predictable integer value, enhancing data security.

Enhancement

API Endpoint Restructuring: All API endpoints have been relocated to their new versioned URLs to ensure proper API versioning and maintain backward compatibility. Refer to the respective API documentation for details.

Upcoming deprecation of OP Events Staging API 0.0.1

Deprecation

OP Events Staging API 0.0.1 will be deprecated as of December 07, 2023 That means that no patch releases will be made, even for critical security issues, after this date. For improved security, and new features, please use the newest version of OP Events Staging API as soon as possible.
